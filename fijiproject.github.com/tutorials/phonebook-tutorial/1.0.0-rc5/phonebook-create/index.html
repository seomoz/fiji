Create a Table

  • ← Previous
  • Next →

We need to create a table to store your bajillion phonebook contacts. Creating a table in Fiji amounts to specifying a layout and registering that layout with Fiji. Layouts can be specified using the Fiji Data Definition Language, DDL, or in a JSON form.

Using the DDL

We assume that you have fiji-schema-shell installed. If not, you should consult the Get Started section of the Fiji website.

We have provided the phonebook layout in the $FIJI_HOME/examples/phonebook/layout.ddl file. For more information about how to create this file, see the DDL Shell Reference.

$FIJI_HOME/schema-shell/bin/fiji-schema-shell \
    --file=$FIJI_HOME/examples/phonebook/layout.ddl

OK.

Verify

Enter the FijiSchema shell using the following command:

$FIJI_HOME/schema-shell/bin/fiji-schema-shell

Use the show tables and describe commands to see your newly created table.

schema> show tables;

Table       Description
=========   ==================================
phonebook   A collection of phone book entries
schema> describe phonebook;

Table: phonebook (A collection of phone book entries)
Row key:
    key: STRING NOT NULL

Column family: info
    Description: basic information

    Column info:firstname (First name)
        Schema: "string"

    Column info:lastname (Last name)
        Schema: "string"

    Column info:email (Email address)
        Schema: "string"

    Column info:telephone (Telephone number)
        Schema: "string"

    Column info:address (Street address)
        Schema: "org.fiji.examples.phonebook.Address"

Column family: derived
    Description: Information derived from an individual's address.

    Column derived:addr1 (Address line one.)
        Schema: "string"

    Column derived:apt (Address Apartment number.)
        Schema: ["string","null"]

    Column derived:addr2 (Address line two.)
        Schema: "string"

    Column derived:city (Address city.)
        Schema: "string"

        Schema: "string"

    Column derived:zip (Address zip code.)
        Schema: "int"

Column family: stats
    Description: Statistics about a contact.

    Column stats:talktime (Time spent talking with this person)
        Schema: (counter)
schema> quit;

Using JSON

A low level way of providing the layout is by using JSON. To learn more about specifying the layout in JSON, take a look at Managing Data.

But first, we need to delete the table we just created, just so that we can create it another time using JSON! Use the following command to delete the table:

$FIJI_HOME/bin/fiji delete --target=fiji://.env/default/phonebook

Are you sure you want to delete Fiji table 'fiji://localhost:2181/default/phonebook/'?
Please answer yes or no.
yes
13/03/13 19:19:28 INFO org.apache.hadoop.hbase.client.HBaseAdmin: Started disable of fiji.default.table.phonebook
13/03/13 19:19:29 INFO org.apache.hadoop.hbase.client.HBaseAdmin: Disabled fiji.default.table.phonebook
13/03/13 19:19:29 INFO org.apache.hadoop.hbase.client.HBaseAdmin: Deleted fiji.default.table.phonebook
Fiji table 'fiji://localhost:2181/default/phonebook/' deleted.

The command below creates the same phonebook table with the layout specified in the layout.json file in your $FIJI_HOME/examples/phonebook directory.

$FIJI_HOME/bin/fiji create-table --table=fiji://.env/default/phonebook \
    --layout=$FIJI_HOME/examples/phonebook/layout.json

Parsing table layout: $FIJI_HOME/examples/phonebook/layout.json
Creating fiji table: fiji://localhost:2181/default/phonebook/...

Verify

To ensure that your table exists, use the fiji ls command to show the available tables in your Fiji instance.

$FIJI_HOME/bin/fiji ls fiji://.env/default

The above command should list phonebook as a table.
