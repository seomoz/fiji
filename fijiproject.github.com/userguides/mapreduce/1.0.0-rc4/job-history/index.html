Job History

  • ← Previous
  • Next →

Motivation

While Hadoop’s job tracker provides detailed information about jobs that have been run in the cluster, it is not a persistent data store for such information.  After installation of the job_history table, Fiji automatically tracks information into the job_history table which can be accessed later. This information includes an xml dump of the full job configuration, start times, end times, and all job counters. /

Setup

The job history tables can be installed into the default Fiji instance with the job-history command:

fiji job-history --fiji=fiji://.env/default --install

If you would like to install the job history table into a different instance, pass in the relevant URI into the –fiji parameter.

You can verify that the table was installed properly using the ls command:

fiji ls --fiji=fiji://.env/default/job_history

Any new MapReduce jobs that are run within Fiji will now save relevant job related metadata into this table.

Classes Overview

The org.fiji.mapreduce.JobHistoryFijiTable is the main class responsible for providing access to the job_history table.  Currently it provides the ability to record and retrieve job metadata. This is a framework-audience class and subject to change between minor versions

Using the API

The JobHistoryFijiTable class surfaces the call getJobDetails for retrieving the recorded metadata.

Jobs that are created using FijiMapReduceJob will automatically record metadata to the job_history table if it has been installed. Any MapReduce jobs using the job builders in Fiji for bulk importers, producers, or gatherers fall under this category and will report data.

Example

The job_history table is a Fiji table under the hood, and can thus be inspected using the fiji ls tool. The EntityId associated with the job_history table is the jobId.  For example, to look at all of the jobIds that have been recorded:

fiji ls --fiji=fiji://.env/default/job_history --columns=info:jobId

There is also a job_history tool, which displays the job history data in a more human readable format.  For example, to look up job data for the job ‘job_20130221123621875_0001’

fiji job-history --fiji=fiji://.env/default --job-id=job_20130221123621875_0001
